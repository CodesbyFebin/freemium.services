What is n8n?

n8n (pronounced "nodemation") is a fair-code licensed workflow automation platform that enables developers and technical teams to connect apps, APIs, and services without writing complex integration code. It's the most powerful self-hostable alternative to Zapier, Make (formerly Integromat), and Tray.io.

Unlike SaaS-only automation tools, n8n can be deployed entirely on your own infrastructure — your VPS, bare metal server, Docker, Kubernetes, or via the TurboQuant edge compute network. When self-hosted, there are no execution limits, no per-operation charges, and complete data sovereignty.

With 47,200+ GitHub stars and a rapidly growing ecosystem, n8n is the de facto standard for developer-grade open-source automation.

Self-Hosting Guide: Deploy n8n with Docker

Self-hosting n8n is straightforward. The official Docker image is maintained and updated with each release. Here's a complete production-ready setup:

1. Quick Start (Single Command)

bash — Quick n8n Deploy
# Pull the official n8n Docker image
$ docker pull n8nio/n8n
latest: Pulling from n8nio/n8n... done
 
# Run n8n with persistent data volume
$ docker run -d \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  n8nio/n8n
a3f8c2d1e4b5...
✓ n8n running at http://localhost:5678

2. Production Setup with Docker Compose + PostgreSQL

# docker-compose.yml — Production n8n Setup version: '3.8' services: n8n: image: n8nio/n8n restart: unless-stopped ports: - "5678:5678" environment: - DB_TYPE=postgresdb - DB_POSTGRESDB_HOST=postgres - DB_POSTGRESDB_PORT=5432 - DB_POSTGRESDB_DATABASE=n8n - DB_POSTGRESDB_USER=n8n - DB_POSTGRESDB_PASSWORD=${POSTGRES_PASSWORD} - N8N_BASIC_AUTH_ACTIVE=true - N8N_BASIC_AUTH_USER=${N8N_USER} - N8N_BASIC_AUTH_PASSWORD=${N8N_PASSWORD} - WEBHOOK_URL=https://your-domain.com/ volumes: - n8n_data:/home/node/.n8n depends_on: - postgres postgres: image: postgres:15-alpine restart: unless-stopped environment: - POSTGRES_USER=n8n - POSTGRES_PASSWORD=${POSTGRES_PASSWORD} - POSTGRES_DB=n8n volumes: - postgres_data:/var/lib/postgresql/data volumes: n8n_data: postgres_data:
bash — Start Production n8n
# Copy .env file and set your passwords
$ cp .env.example .env && nano .env
 
# Start n8n + PostgreSQL
$ docker-compose up -d
Creating network "n8n_default"
Creating n8n_postgres_1 ... done
Creating n8n_n8n_1 ... done
✓ n8n available at http://localhost:5678

3. Add a Reverse Proxy (Nginx/Caddy)

For production with SSL, use Caddy (easiest) or Nginx:

# Caddyfile — Auto SSL with Let's Encrypt n8n.yourdomain.com { reverse_proxy localhost:5678 }
bash — Deploy Caddy
$ sudo apt install caddy
$ sudo caddy run --config /etc/caddy/Caddyfile
✓ SSL certificate issued. n8n live at https://n8n.yourdomain.com

Frequently Asked Questions

Is n8n free to use?+
Yes. n8n has two usage modes: (1) Self-hosted — completely free with no execution limits, full feature access, and no credit card required. (2) n8n Cloud — free trial, then $20/month for a starter plan. When you self-host n8n, you only pay for your server costs (a $6/mo Hetzner VPS is sufficient for most use cases).
How do I self-host n8n?+
The easiest way is with Docker. Run: docker run -d --name n8n -p 5678:5678 -v ~/.n8n:/home/node/.n8n n8nio/n8n. For production, use the Docker Compose setup with PostgreSQL described in our guide above. For cloud deployment, Hetzner is the most cost-effective option. TurboQuant's edge network also supports n8n deployment at significantly reduced cost.
What is n8n fair-code license?+
n8n uses a "fair-code" license, which means: (1) The source code is publicly available on GitHub. (2) You can use it for free for personal use, internal business use, and commercial projects. (3) You cannot offer n8n as a managed SaaS service to third parties without a commercial license. For almost all use cases — including building AI products and internal tools — n8n is effectively open-source and free.
Can n8n integrate with Claude and other LLMs?+
Yes. n8n has native integrations for: Anthropic Claude (direct API node), OpenAI (GPT-4, embeddings), Mistral AI, Cohere, HuggingFace Inference API, and Ollama (local models). You can chain LLM calls, build RAG pipelines with vector stores, and create autonomous agents — all visually in n8n's workflow editor.
What are the system requirements for self-hosting n8n?+
Minimum: 1 vCPU, 1GB RAM (SQLite). Recommended production: 2 vCPU, 4GB RAM, PostgreSQL. For heavy workloads with many concurrent workflows: 4 vCPU, 8GB RAM. n8n runs great on a €3.99/mo Hetzner CAX11 instance. For AI-heavy workflows, consider TurboQuant's edge nodes with dedicated GPU access.
How does n8n compare to Zapier for AI workflows?+
n8n is significantly more powerful for AI workflows. Unlike Zapier, n8n offers: native LLM nodes (Claude, GPT-4, Ollama), vector store integration (Qdrant, Pinecone, Weaviate), AI agent frameworks with memory and tool use, and Python/JavaScript code nodes for custom logic. Plus n8n is self-hostable — meaning no per-execution costs and no data leaving your infrastructure. For developers building AI-powered automation, n8n is the clear choice.
