Quickstart: Essential Self-Hosted Stack

Get a complete self-hosted AI + automation stack running in under 10 minutes with Docker Compose. This stack powers hundreds of teams on a single €6/mo VPS.

bash — Full AI Self-Hosted Stack
# 1. Install Docker + Compose on Ubuntu
$ curl -fsSL https://get.docker.com | sh
✓ Docker 26.x installed
 
# 2. Clone the Freemium.Services starter stack
$ git clone https://github.com/freemium-services/starter-stack
$ cd starter-stack && cp .env.example .env
 
# 3. Configure your domain + passwords in .env
$ nano .env
 
# 4. Start the full stack (n8n + Dify + Qdrant + Caddy)
$ docker-compose up -d
✓ qdrant:latest → :6333
✓ n8nio/n8n → :5678
✓ langgenius/dify → :3000
✓ caddy (SSL) → :443
 
✓ Stack live at https://your-domain.com

Self-Hosting FAQ

What does self-hosting mean?+
Self-hosting means running software on your own server or infrastructure rather than using a vendor's managed cloud service. You take an open-source tool, deploy it on a VPS, home server, or cloud VM, and you have full control over the data, configuration, and uptime. The software is the same — you just run it yourself instead of paying the vendor to run it for you.
What server do I need to self-host AI tools?+
For most AI tools (n8n, Dify, Onyx, Qdrant), a Hetzner CAX11 (2 ARM vCPU, 4GB RAM, €3.99/mo) or Hetzner CX22 (2 vCPU, 4GB RAM, €4.49/mo) is sufficient. For running local LLMs with Ollama, you need more RAM (16GB+ for 7B models, 32GB for 13B models). For GPU inference, consider TurboQuant's edge nodes or a dedicated GPU server from vast.ai.
How do I get SSL/HTTPS for self-hosted apps?+
Use Caddy as your reverse proxy — it automatically provisions and renews Let's Encrypt SSL certificates with a single config line: your-domain.com { reverse_proxy localhost:PORT }. Caddy is the easiest option. Alternatively, use Nginx Proxy Manager (GUI-based) or Traefik for Docker-based setups. You'll need a domain name pointed to your server's IP via DNS A record.
Is self-hosting secure?+
Self-hosting can be highly secure if configured correctly. Key practices: keep Docker images updated, use strong passwords and environment variables (not hardcoded), enable firewall (UFW), use SSL/TLS, restrict ports (only expose 80/443), enable fail2ban, and use SSH key authentication. Many self-hosted tools are inherently more secure than SaaS because your data never leaves your infrastructure.
Can I self-host on a Raspberry Pi or home server?+
Yes, many tools run well on ARM hardware including Raspberry Pi. Tools like n8n, Vaultwarden, Uptime Kuma, Meilisearch, and Qdrant all have ARM Docker images. For heavy AI workloads (Dify, Onyx with local LLMs), you'll need more RAM. A Raspberry Pi 5 (8GB) can handle lightweight self-hosted stacks. Use a VPS for production workloads where uptime matters.
How does TurboQuant help with self-hosting?+
TurboQuant (turboquant.network) is a DePIN edge compute network that provides distributed infrastructure for self-hosted AI workloads. Instead of managing your own VPS, you can deploy to TurboQuant's edge nodes — distributed globally, optimized for AI inference, and typically 5-10x cheaper than AWS for GPU-accelerated workloads. TurboQuant also provides managed orchestration for complex multi-container self-hosted stacks.
